# MoonBit GPT Edge Demo

A Transformer implementation written entirely in [MoonBit](https://www.moonbitlang.com/), compiled to WebAssembly (wasm-gc) and running fully in the browser. No server, no Python, no GPU.

![demo](docs/demo-arch.gif)

[日本語版 README](README.ja.md)

## Features

- **Two architectures switchable at runtime**
  - **Enc-Dec** — original Transformer (Vaswani 2017): bidirectional encoder + autoregressive decoder with cross-attention
  - **GPT** — decoder-only language model: causal self-attention, next-token prediction
- **From-scratch autograd engine** in MoonBit — no external ML library
- **AdamW optimizer** with linear learning-rate decay
- **Live visualization panels** — attention matrices, Q/K/V strips, MLP activations, embedding PCA (3D), hidden state trajectories, next-token predictions
- **Ablation experiments** — toggle Skip connections, Positional Embeddings, and Attention on/off at runtime
- **Runs on the edge** — compiled to WebAssembly via `moon build --target wasm-gc`, served as a static page
- **React + Vite frontend** — TypeScript, floating draggable/resizable panels

## Architecture

```
src/
  autograd.mbt    — Value struct, forward/backward ops (Add, Mul, exp, log, relu, …)
  config.mbt      — Hyperparameters, dataset (50 English recipe sentences)
  globals.mbt     — Global Ref state: weights, Adam moments, viz buffers, PCA state
  init.mbt        — Weight initialization (Gaussian), ablation flags
  ops.mbt         — linear, softmax, rmsnorm (autograd + float variants)
  forward.mbt     — encoder_forward, decoder_step (autograd)
  forward_f.mbt   — encoder_forward_f, decoder_step_f (float, for visualization)
  train.mbt       — train_step (AdamW), generate, sample_token
  viz.mbt         — run_attn_viz, run_network_viz, run_residual_viz
  draw.mbt        — canvas rendering via display list protocol (tick_pca, draw_arch_diagram, …)
cmd/main/
  main.mbt        — pub fn WASM exports (mbt_init, mbt_train_step, …)
  moon.pkg        — wasm-gc export list (DCE guard)
web/
  public/
    moonbit.wasm  — compiled output (generated by make build-mbt)
  src/
    mbt/
      ffi.ts          — WebAssembly loader + typed wrappers (mbt.init(), mbt.trainStep(), …)
    hooks/
      useTraining.ts  — rAF-based training loop with cleanup
      usePcaCanvas.ts — PCA animation loop + mouse/wheel interaction
    canvas/
      displayList.ts  — display list executor (CMD constants, 36 commands)
      drawAttn.ts     — attention/embedding canvas helpers
      drawLossChart.ts
    components/
      FloatingPanel.tsx   — draggable + resizable panel base
      LiveBadge.tsx       — "live" indicator shown during training
      Toolbar.tsx         — top bar + sidebar
      panels/             — TrainingPanel, GenPanel, AttnPanel, EmbPanel,
                            EmbPcaPanel, HiddenPanel, MlpPanel, PredPanel,
                            DatasetPanel
    App.tsx
```

**Model size** (defaults): `n_layer=1  n_embd=32  n_head=4  block_size=8`

## WebAssembly FFI

MoonBit is compiled with `--target wasm-gc`. Functions are exported as WASM exports and called directly from JavaScript via `WebAssembly.instantiateStreaming`.

Since `String` in wasm-gc is a UTF-16 array that cannot cross the WASM boundary directly, string I/O uses an index-access protocol:

- **JS → MoonBit**: `mbt_in_str_clear()` + `mbt_in_str_push(charCode)` per character
- **MoonBit → JS**: call getter (e.g. `mbt_get_viz_tokens()`), then read with `mbt_str_len()` / `mbt_str_char(i)`
- **Array[Double]**: same pattern via `mbt_arr_len()` / `mbt_arr_at(i)`

## Getting Started

### Prerequisites

- [MoonBit toolchain](https://www.moonbitlang.com/download/) (`moon` CLI)
- Node.js 18+ and npm

### Build & Run

```bash
# Install frontend dependencies (first time only)
cd web && npm install && cd ..

# Build MoonBit → wasm + Vite → dist/
make build

# Development: MoonBit build + Vite dev server (hot reload)
make dev
# → open http://localhost:5173

# Preview production build
make serve
```

### Test

```bash
moon test
```

43 tests cover the autograd engine (including finite-difference gradient checks), neural network operators (rmsnorm, linear, softmax), training convergence, text generation, custom datasets, and ablation flags.

## Usage

1. Select **Enc-Dec** or **GPT** mode with the toggle buttons in the toolbar.
2. Press **Start** in the sidebar — the **Embedding PCA** and **Pos Embedding Path** panels open automatically.
3. Click **Analyze** to visualize the current weights on the phrase in the input box.
4. Open additional panels from the sidebar: **Loss / PCA / PosEmb / Data**.

### Ablation Experiments

Toggle switches in the sidebar disable individual components mid-training. The model resets and reinitializes automatically.

| Switch | Effect |
|--------|--------|
| **Attn: OFF** | Loss converges more slowly; attention matrices become zero |
| **Skip: OFF** | No residual connections; gradient flow degrades |
| **PosEmb: OFF** | No positional information; attention becomes position-invariant |

### Visualization Panels

| Panel | Description |
|-------|-------------|
| Loss | Training loss curve (cross-entropy) |
| Attention Weights | Per-head attention matrices (4 heads) |
| Token Embedding Similarity | Cosine similarity heatmap across vocabulary |
| Embedding PCA | 3D PCA projection of word embeddings — rotate/pan/zoom |
| Pos Embedding Path | Residual stream trajectory through PCA space |
| Hidden State Evolution | Encoder hidden states across layers |
| MLP Activations | FC1 ReLU activation values |
| Next-word Predictions | Top-k logits per position |
| Training Dataset | View and replace the training corpus |

## How It Works

### Training

| Mode | Objective | Encoder |
|------|-----------|---------|
| Enc-Dec | Given first half of sentence → predict second half | Bidirectional self-attention |
| GPT | Given token at position t → predict token at t+1 | None (decoder only) |

Backward pass is handled by a hand-rolled autograd engine: each `Value` node stores its children and local gradients; `v_backward` runs topological-order reverse accumulation.

### PCA Visualization

Embeddings are projected onto 3 principal components via power iteration. Sign ambiguity (a known issue with eigenvectors) is resolved by aligning each new eigenvector's sign against the previous frame using a dot-product check, preventing sudden flips during training.

### Display List Protocol

MoonBit generates canvas drawing commands as a flat `Array[Double]` (command ID followed by arguments). JavaScript interprets and executes these using `executeDisplayList`. This keeps all rendering logic in MoonBit while remaining decoupled from the DOM.
