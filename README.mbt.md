# MoonBit GPT Edge Demo

A Transformer implementation written entirely in [MoonBit](https://www.moonbitlang.com/), compiled to WebAssembly (wasm-gc) and running fully in the browser. No server, no Python, no GPU.

## Features

- **Two architectures switchable at runtime**
  - **Enc-Dec** — original Transformer (Vaswani 2017): bidirectional encoder + autoregressive decoder with cross-attention
  - **GPT** — decoder-only language model: causal self-attention, next-token prediction
- **From-scratch autograd engine** in MoonBit — no external ML library
- **AdamW optimizer** with linear learning-rate decay
- **Live architecture visualization** — attention matrices, Q/K/V strips, residual connections, KV-cache arrows, all drawn on canvas
- **Runs on the edge** — compiled to WebAssembly via `moon build --target wasm-gc`, served as a static page
- **React + Vite frontend** — TypeScript, component-based UI

## Architecture

```
src/
  autograd.mbt   — Value struct, forward/backward ops (Add, Mul, exp, log, relu, …)
  config.mbt     — Hyperparameters, dataset (50 English recipe sentences)
  globals.mbt    — Global Ref state: weights, Adam moments, viz buffers, gpt_mode flag
  init.mbt       — Weight initialization (Gaussian), set_gpt_mode / get_gpt_mode
  ops.mbt        — linear, softmax, rmsnorm (autograd + float variants)
  forward.mbt    — encoder_forward, decoder_step (autograd)
  forward_f.mbt  — encoder_forward_f, decoder_step_f (float, for visualization)
  train.mbt      — train_step (AdamW), generate, sample_token
  viz.mbt        — run_attn_viz, run_network_viz, run_network_viz_gpt
cmd/main/
  main.mbt       — pub fn exports (mbt_init, mbt_train_step, …)
  moon.pkg       — wasm-gc export list (DCE guard)
web/
  public/
    moonbit.wasm — compiled output (generated by make build-mbt)
  src/
    mbt/moonbit.ts     — WebAssembly loader + typed wrapper (mbt.init(), mbt.trainStep(), …)
    hooks/useTraining.ts — rAF-based training loop hook
    canvas/            — drawArchDiagram, drawAttn, drawLossChart (pure TS functions)
    components/        — Toolbar, Sidebar, FloatingPanel, panels
    App.tsx
```

**Model size** (defaults): `n_layer=1  n_embd=32  n_head=4  block_size=8`

### WebAssembly FFI

MoonBit is compiled with `--target wasm-gc`. Functions are exported as WASM exports and called directly from JavaScript via `WebAssembly.instantiateStreaming`.

Since `String` in wasm-gc is a UTF-16 array that cannot cross the WASM boundary directly, string I/O uses an index-access protocol:
- **JS → MoonBit**: `mbt_in_str_clear()` + `mbt_in_str_push(charCode)` per character
- **MoonBit → JS**: call getter (e.g. `mbt_get_viz_tokens()`), then read with `mbt_str_len()` / `mbt_str_char(i)`
- **Array[Double]**: same pattern via `mbt_arr_len()` / `mbt_arr_at(i)`

## Getting Started

### Prerequisites

- [MoonBit toolchain](https://www.moonbitlang.com/download/) (`moon` CLI)
- Node.js 18+ and npm

### Build & Run

```bash
# Install frontend dependencies (first time only)
cd web && npm install && cd ..

# Build MoonBit → wasm + Vite → dist/
make build

# Development: MoonBit build + Vite dev server (hot reload)
make dev
# → open http://localhost:5173

# Preview production build
make serve
```

### Usage

1. Select **Enc-Dec** or **GPT** mode with the toggle buttons in the toolbar.
2. Press **Start** (sidebar) to begin training (2000 steps, AdamW).
3. Click **Analyze** to visualize the current weights on the phrase in the input box.
4. Use the side panel buttons (**Loss / Gen / Attn / Emb / Hidden / MLP / Pred**) to open floating panels.

## How It Works

### Training

| Mode | Objective | Encoder |
|------|-----------|---------|
| Enc-Dec | Given first half of sentence → predict second half | Bidirectional self-attention |
| GPT | Given token at position t → predict token at t+1 | None (decoder only) |

Backward pass is handled by a hand-rolled autograd engine: each `Value` node stores its children and local gradients; `v_backward` runs topological-order reverse accumulation.

### Visualization

The canvas diagram updates every 5 training steps and shows:

- Token & positional embeddings
- Per-head Q / K / V strips and full attention matrices
- Residual (skip) connections
- A×V head outputs before concat+linear
- KV-cache flow arrows between decoder columns (indigo)
- Encoder→decoder K,V bridge (orange, Enc-Dec mode only)
